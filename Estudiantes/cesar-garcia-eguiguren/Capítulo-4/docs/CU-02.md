# CU-02 — Listar Empleados

## Descripción funcional

La pantalla de empleados permite al actor consultar el directorio completo de empleados activos de la organización. El listado se adapta automáticamente al rol del usuario: el Director ve todos los empleados del sistema, mientras que el Responsable únicamente ve los empleados que pertenecen a su ámbito organizativo, calculado en el momento del inicio de sesión y embebido en la sesión.

Esta vista actúa como punto de entrada para acceder al resumen individual de cualquier empleado (CU-03).

## Qué puede hacer el usuario

### Filtros disponibles

| Filtro | Tipo | Descripción |
|---|---|---|
| **Búsqueda por nombre** | Campo de texto | Filtra en tiempo real los empleados cuyo nombre contiene el texto introducido. El sistema espera 300 ms tras el último carácter antes de lanzar la consulta (debounce). |
| **Departamento** | Selector desplegable | Restringe el listado a los empleados del departamento seleccionado. Solo aparecen los departamentos accesibles según el ámbito del actor. |
| **Mostrar solo activos** | Casilla de verificación | Activada por defecto. Al desmarcarla, se incluyen también empleados dados de baja en el ERP. |
| **Limpiar filtros** | Botón | Restablece todos los filtros a su estado inicial y vuelve a la primera página. |

### Ordenación

El actor puede hacer clic sobre cualquier cabecera de columna que tenga el indicador de ordenación para cambiar el criterio. Al hacer clic sobre la misma columna, alterna entre orden ascendente y descendente. La ordenación se ejecuta en el servidor, por lo que afecta a todo el conjunto de resultados, no solo a la página visible.

| Columna ordenable | Criterio |
|---|---|
| Empleado | Orden alfabético por nombre |
| Departamento | Orden alfabético por nombre de departamento |
| Cargo | Orden alfabético por título de puesto |
| Coste/h | Orden numérico por coste por hora |

### Paginación

El listado muestra 25 empleados por página. El actor puede navegar entre páginas mediante los controles de paginación situados al final de la tabla. El sistema indica el total de registros y la página actual en el encabezado de la vista.

### Navegación al detalle

Al hacer clic sobre cualquier fila de la tabla, el sistema navega a la pantalla de resumen del empleado seleccionado (CU-03).

## Datos mostrados por empleado

| Campo | Descripción |
|---|---|
| **Nombre** | Nombre completo del empleado en el ERP. Enlace clicable al detalle. |
| **Departamento** | Nombre del departamento al que pertenece. |
| **Cargo** | Título del puesto de trabajo. |
| **Email** | Dirección de correo corporativo. |
| **Coste/h** | Coste horario del empleado en euros, usado en los cálculos de rentabilidad. |
| **Estado** | Indicador visual (badge) que muestra si el empleado está activo o inactivo. |

---

## Restricciones de acceso

- **Director:** ve todos los empleados del sistema sin restricción de ámbito.
- **Responsable:** ve únicamente los empleados bajo su gestión directa o indirecta, según los departamentos y proyectos que gestiona.
- Cualquier intento de acceder con filtros fuera del ámbito (por ejemplo, un departamento que no gestiona) devuelve un error 403 sin mostrar datos.

## Implementación

> **Archivos implicados:** `app/routes/employee.py` · `app/services/employee_service.py` · `app/repositories/employee.py`

### Capa de rutas — `routes/employee.py`

El endpoint declara todos los parámetros de filtrado como `Query` opcionales. FastAPI los valida automáticamente y los inyecta en la función. El guard `require_manager_or_above` extrae y verifica el JWT antes de que el handler se ejecute.

```python
@router.get("/", response_model=PaginatedResponse[EmployeeDetail])
def list_employees(
    department_id: Optional[int] = Query(None),
    search:        Optional[str] = Query(None),
    active:        Optional[bool] = Query(None),
    # Columnas permitidas para ordenar; el patrón regex evita inyecciones
    sort_by:    str = Query("name", pattern="^(name|department_name|job_title|hourly_cost)$"),
    sort_order: str = Query("asc",  pattern="^(asc|desc)$"),
    page:      int = Query(1,  ge=1),
    page_size: int = Query(50, ge=1, le=200),
    db: Session      = Depends(get_db),
    cu: CurrentUser  = Depends(require_manager_or_above),  # 403 si rol=empleado
):
    return EmployeeService(db).list_employees(
        cu,
        department_id=department_id, search=search, active=active,
        sort_by=sort_by, sort_order=sort_order,
        page=page, page_size=page_size,
    )
```

### Capa de servicio — `services/employee_service.py`

El servicio aplica la lógica de **scope**: si el actor es Responsable y no ha filtrado por departamento, se restringe el conjunto a `cu.employee_ids` (lista embebida en el JWT durante el login).

```python
def list_employees(self, cu, department_id=None, search=None, active=None,
                   sort_by="name", sort_order="asc", page=1, page_size=50):
    if department_id:
        # Comprueba que el departamento existe y pertenece al ámbito del actor
        verify_department_exists(self.db, department_id)
        verify_department_scope(cu, department_id)  # 403 si no está en cu.department_ids

    # Si es Responsable sin filtro de dpto, limita a sus empleados directos
    emp_ids = cu.employee_ids if cu.is_responsable and not department_id else None

    query = get_filtered_employees_query(
        self.db,
        employee_ids=emp_ids,
        department_id=department_id,
        search=search, active=active,
        sort_by=sort_by, sort_order=sort_order,
    )
    return paginate(query, page, page_size)   # devuelve PaginatedResponse
```

### Capa de repositorio — `repositories/employee.py`

Construye la consulta SQLAlchemy de forma incremental. El `outerjoin` con `Department` permite ordenar y filtrar por nombre de departamento sin perder empleados sin departamento asignado.

```python
def get_filtered_employees_query(db, employee_ids=None, department_id=None,
                                  search=None, active=None,
                                  sort_by="name", sort_order="asc"):
    # Base: empleados activos + join con Department para obtener department_name
    query = _base_query(db).filter(Employee.active == True)

    if employee_ids is not None:
        query = query.filter(Employee.id.in_(employee_ids))   # scope Responsable
    if department_id:
        query = query.filter(Employee.department_id == department_id)
    if active is not None:
        query = query.filter(Employee.active == active)        # permite ver inactivos
    if search:
        # Búsqueda insensible a mayúsculas/acentos mediante ILIKE de PostgreSQL
        query = query.filter(Employee.name.ilike(f"%{search}%"))

    order_fn = desc if sort_order.lower() == "desc" else asc
    if sort_by == "department_name":
        # Orden por columna del join; nullslast() coloca sin-dpto al final
        query = query.order_by(order_fn(Department.name).nullslast())
    elif hasattr(Employee, sort_by):
        query = query.order_by(order_fn(getattr(Employee, sort_by)).nullslast())

    return query   # el servicio aplica paginate() sobre este query
```
