# CU-08 — Listar tareas

## Descripción funcional

La pantalla de tareas centraliza el acceso a todas las tareas del sistema a través de un único endpoint polimórfico: `GET /tasks/filter`. El mismo endpoint es reutilizado por las pestañas de la ficha de empleado (CU-03), las pestañas del detalle de proyecto y la página global de tareas, variando únicamente los parámetros de filtrado.

El modo de construcción de los ítems de respuesta se determina automáticamente por la combinación de `status` y `employee_id`:

| `status` | `employee_id` | Tipo de ítem devuelto |
|---|---|---|
| `pending` | presente | `PendingTaskItem` |
| `completed` | presente | `CompletedTaskItem` |
| cualquier otro | presente | `AssignedTaskItem` |
| — | ausente | `TaskResponse` (modo genérico) |

## Qué puede hacer el usuario

### Filtros disponibles

| Filtro | Parámetro | Descripción |
|---|---|---|
| **Estado** | `status` | `pending`, `completed` o `overdue`. Filtra por etapa abierta/cerrada. |
| **Etapa** | `stage_id` | Etapa concreta del flujo Kanban. |
| **Proyecto** | `project_id` | Restringe al proyecto seleccionado. Aplica validación de scope si el actor es Responsable. |
| **Empleado** | `employee_id` | Tareas asignadas a ese empleado. Aplica validación de scope. |
| **Responsable** | `responsable` | Si `true`, devuelve las tareas donde el empleado es responsable (`user_id`). |
| **Fecha de fin** | `date_from` / `date_to` | Rango sobre `date_deadline`. |
| **Fecha de asignación** | `date_assign` | Filtra tareas asignadas a partir de esa fecha. |
| **Solo raíz** | `root_only` | Si `true`, excluye las subtareas (`parent_id IS NULL`). |
| **Búsqueda** | `search` | Filtra por nombre de tarea (ILIKE). |

### Ordenación y paginación

- Columna de ordenación configurable mediante `sort_by` y `sort_order`.
- Paginación server-side: `page` y `page_size` (máx. 200 por página, por defecto 50).

---

## Datos mostrados (modo genérico `TaskResponse`)

| Campo | Descripción |
|---|---|
| **Nombre** | Nombre de la tarea en Odoo. |
| **Proyecto** | Proyecto padre. |
| **Etapa** | Nombre traducido de la etapa Kanban. |
| **Horas planificadas** | `planned_hours` del modelo ORM. |
| **Horas trabajadas** | Calculadas mediante `get_worked_hours_batch` sobre `account_analytic_line`. |
| **Fecha límite** | `date_deadline`. |
| **Cerrada** | Booleano calculado a partir de la etapa (`is_closed`). |

---

## Campos adicionales por modo

### `PendingTaskItem`
Añade `pending_hours` (= `planned - worked`), `is_overdue` y `date_assign`.

### `CompletedTaskItem`
Añade `actual_hours`, `productivity` (= `planned / actual × 100`), `completion_date` y `on_time`.

### `AssignedTaskItem`
Combina los campos de ambos modos: `pending_hours`, `actual_hours`, `productivity`, `is_closed` e `is_overdue`.

---

## Restricciones de acceso

- **Director:** ve tareas de todos los proyectos del sistema.
- **Responsable:** si no se especifica `employee_id` ni `project_id`, el servicio restringe automáticamente a `cu.project_ids`. Si se especifica `project_id`, se comprueba que pertenezca al ámbito del Responsable (403 si no). Si se especifica `employee_id`, se comprueba que pertenezca a `cu.employee_ids`.

### Capa de rutas — `routes/task.py`

El endpoint agrega todos los filtros en un único handler. El parámetro `status` lleva un `pattern` regex que previene valores no contemplados.

```python
# router prefix: /tasks
@router.get("/filter")
def filter_tasks(
    status:       Optional[str]  = Query(None, pattern="^(pending|completed|overdue)$"),
    stage_id:     Optional[int]  = Query(None),
    project_id:   Optional[int]  = Query(None),
    department_id:Optional[int]  = Query(None),
    date_from:    Optional[date] = Query(None),
    date_to:      Optional[date] = Query(None),
    search:       Optional[str]  = Query(None),
    root_only:    bool           = Query(False),
    date_assign:  Optional[date] = Query(None),
    employee_id:  Optional[int]  = Query(None),
    responsable:  Optional[bool] = Query(None),
    sort_by:      str            = Query("date_deadline"),
    sort_order:   str            = Query("asc"),
    page:         int            = Query(1,  ge=1),
    page_size:    int            = Query(50, ge=1, le=200),
    db: Session     = Depends(get_db),
    cu: CurrentUser = Depends(require_manager_or_above),
):
    return TaskService(db).filter_tasks(
        cu, status=status, stage_id=stage_id,
        project_id=project_id, department_id=department_id,
        date_from=date_from, date_to=date_to, search=search,
        root_only=root_only, date_assign=date_assign,
        employee_id=employee_id, responsable=responsable,
        sort_by=sort_by, sort_order=sort_order,
        page=page, page_size=page_size,
    )
```

### Capa de servicio — `services/task_service.py`

`filter_tasks` aplica la lógica de scope antes de construir la query. Si el actor es Responsable, `effective_project_ids` limita el conjunto a `cu.project_ids`. Tras paginar, `get_worked_hours_batch` resuelve las horas reales en **una sola consulta SQL** para todos los items de la página (evita N+1). Finalmente, `_build_items` selecciona el tipo de objeto según la combinación `status` / `employee_id`.

```python
def filter_tasks(self, cu, status=None, stage_id=None, project_id=None,
                 department_id=None, date_from=None, date_to=None, search=None,
                 root_only=False, date_assign=None, employee_id=None,
                 responsable=None, sort_by="date_deadline", sort_order="asc",
                 page=1, page_size=50):

    # ── Scope del Responsable ──────────────────────────────────────────────
    effective_project_ids = None
    effective_project_id  = project_id

    if cu.is_responsable and employee_id is None:
        if project_id and project_id not in cu.project_ids:
            raise HTTPException(status_code=403, detail="Acceso denegado al proyecto")
        # Restringe a los proyectos del actor; si hay project_id lo incluye como lista
        effective_project_ids = [project_id] if project_id else cu.project_ids
        effective_project_id  = None  # ya está contenido en effective_project_ids

    elif project_id:
        verify_project_exists(self.db, project_id)

    if employee_id is not None:
        verify_employee_exists(self.db, employee_id)
        if cu.is_responsable and employee_id not in cu.employee_ids:
            raise HTTPException(status_code=403, detail="Acceso denegado al empleado")

    if department_id:
        verify_department_exists(self.db, department_id)
        if cu.is_responsable and department_id not in cu.department_ids:
            raise HTTPException(status_code=403, detail="Acceso denegado al departamento")

    # ── Consulta base y paginación ────────────────────────────────────────
    query = build_filtered_query(
        self.db,
        project_ids=effective_project_ids,
        project_id=effective_project_id,
        stage_id=stage_id, status=status,
        employee_id=employee_id, responsable=responsable,
        department_id=department_id, date_assign=date_assign,
        root_only=root_only, date_from=date_from, date_to=date_to,
        search=search, sort_by=sort_by, sort_order=sort_order,
    )
    result = paginate(query, page, page_size)

    # Una sola consulta SQL para las horas de toda la página (anti N+1)
    hours_map = get_worked_hours_batch(self.db, [t.id for t in result["items"]])
    items = self._build_items(result["items"], status, employee_id, responsable, hours_map)

    return PaginatedResponse(
        items=items, total=result["total"],
        page=result["page"], page_size=result["page_size"],
        total_pages=result["total_pages"],
    )

def _build_items(self, tasks, status, employee_id, responsable, hours_map):
    # El tipo de ítem depende de la combinación status × employee_id
    use_pending   = status == "pending"   and employee_id is not None
    use_completed = status == "completed" and employee_id is not None
    use_assigned  = employee_id is not None and status not in ("pending", "completed")

    items = []
    for task in tasks:
        pn     = extract_translated_name(task.project.name) if task.project else None
        sn     = extract_translated_name(task.stage.name)   if task.stage   else None
        worked = hours_map.get(task.id, 0.0)

        if use_pending:
            items.append(self._to_pending(task, pn, sn, worked))
            # PendingTaskItem: añade pending_hours, is_overdue, date_assign
        elif use_completed:
            items.append(self._to_completed(task, pn, sn, worked))
            # CompletedTaskItem: añade actual_hours, productivity, on_time
        elif use_assigned or responsable:
            items.append(self._to_assigned(task, pn, sn, worked))
            # AssignedTaskItem: combina pending_hours + actual_hours + productivity
        else:
            items.append(TaskResponse(
                id=task.id, name=task.name, project_id=task.project_id,
                parent_id=task.parent_id, planned_hours=task.planned_hours,
                effective_hours=round(worked, 2),
                date_assign=task.date_assign.date() if task.date_assign else None,
                date_deadline=task.date_deadline, date_end=task.date_end,
                is_closed=task.is_closed, stage_name=sn, subtasks=[],
            ))
    return items
```

### Capa de repositorio — `repositories/task.py`

`build_filtered_query` es la función central del módulo de tareas. Usa dos subqueries de etapas (`open_stage_ids_subq` / `closed_stage_ids_subq`) para filtrar por `status` sin depender de nombres de etapa. El filtro por `employee_id` hace un JOIN con `task_assignment` → `hr_employee`. La ordenación se delega en `_apply_task_sort`.

```python
def open_stage_ids_subq(db):
    # Subconsulta reutilizable: IDs de etapas con closed = False
    return db.query(TaskStage.id).filter(TaskStage.closed == False).subquery()

def closed_stage_ids_subq(db):
    return db.query(TaskStage.id).filter(TaskStage.closed == True).subquery()

def build_filtered_query(db, *, project_ids=None, project_id=None,
                          stage_id=None, status=None, employee_id=None,
                          responsable=None, department_id=None, date_assign=None,
                          root_only=False, date_from=None, date_to=None,
                          search=None, sort_by="date_deadline", sort_order="asc"):

    query = db.query(Task).filter(Task.active == True)

    # Scope de proyectos (lista del Responsable o proyecto concreto)
    if project_ids is not None:
        query = query.filter(
            Task.project_id.in_(project_ids) if project_ids else Task.project_id.is_(None)
        )
    elif project_id:
        query = query.filter(Task.project_id == project_id)

    # Filtro por status usando subqueries de etapas (no depende de nombres)
    if stage_id:
        query = query.filter(Task.stage_id == stage_id)
    elif status == "pending":
        query = query.filter(Task.stage_id.in_(open_stage_ids_subq(db)))
    elif status == "completed":
        query = query.filter(Task.stage_id.in_(closed_stage_ids_subq(db)))
    elif status == "overdue":
        query = query.filter(
            Task.stage_id.in_(open_stage_ids_subq(db)),
            Task.date_deadline < date.today(),
        )

    # Filtro por empleado: join task_assignment → hr_employee
    if employee_id is not None:
        if responsable is True:
            query = query.filter(Task.responsable_id == employee_id)
        else:
            query = (query
                     .join(TaskAssignment, Task.id == TaskAssignment.task_id)
                     .join(Employee, Employee.user_id == TaskAssignment.user_id)
                     .filter(Employee.id == employee_id))

    if root_only:
        query = query.filter(Task.parent_id.is_(None))  # excluye subtareas
    if search:
        query = query.filter(Task.name.ilike(f"%{search}%"))

    # Join final con Project para permitir ordenar por project_name
    query = query.outerjoin(Project, Task.project_id == Project.id)
    return _apply_task_sort(query, sort_by, sort_order)
```

`get_worked_hours_batch` (en `repositories/timesheet.py`) resuelve el problema N+1 agrupando todas las sumas de horas en una consulta única para los IDs recibidos:

```python
# repositories/timesheet.py
def get_worked_hours_batch(db: Session, task_ids: list[int]) -> dict[int, float]:
    if not task_ids:
        return {}
    rows = (
        db.query(
            Timesheet.task_id,
            func.coalesce(func.sum(Timesheet.unit_amount), 0).label("hours"),
        )
        .filter(Timesheet.task_id.in_(task_ids))
        .group_by(Timesheet.task_id)
        .all()
    )
    return {r.task_id: float(r.hours) for r in rows}
```
