# CU-13 — Consultar rentabilidad financiera

## Descripción funcional

La pantalla de rentabilidad financiera es **exclusiva del Director**. Muestra el balance global de ingresos, gastos y neto del periodo seleccionado, calculado desde las líneas analíticas de Odoo (`account_analytic_line`). El resumen global se complementa con un desglose tabulado por proyecto y por cliente, y con gráficos de barras y de tarta que visualizan la distribución de rentabilidad.

Desde la tabla de detalle por proyecto o por cliente, el actor puede navegar al drill-down de líneas analíticas (CU-14).

## Qué puede hacer el usuario

### Filtros disponibles

| Filtro | Parámetro | Descripción |
|---|---|---|
| **Fecha de inicio** | `date_from` | Inicio del periodo de análisis. |
| **Fecha de fin** | `date_to` | Fin del periodo de análisis. |
| **Proyecto** | `project_id` | Restringe el desglose a un proyecto concreto (solo en vista por proyecto). |
| **Responsable** | `manager_id` | Filtra los proyectos por su responsable. |

### Vistas disponibles

El actor puede alternar entre tres vistas mediante pestañas:

| Vista | Endpoint | Descripción |
|---|---|---|
| **Resumen global** | `GET /metrics/profitability/summary` | Totales de ingresos, gastos, neto, horas y `profitability_pct` global. Incluye recuento de proyectos por estado. |
| **Por proyecto** | `GET /metrics/profitability/per-project` | Desglose fila por fila con `income`, `expense`, `net`, `total_hours`, `profitability_pct` y estado (`ganancia` / `neutro` / `pérdida`). |
| **Por cliente** | `GET /metrics/profitability/per-client` | Agrupación de proyectos por cliente. Muestra `project_count` y los mismos totales financieros. |

### Drill-down a líneas analíticas (CU-14)

Al pulsar **Ver detalles** sobre una fila de la tabla, el frontend navega al detalle de líneas analíticas del proyecto o cliente seleccionado:
- Por proyecto: `GET /metrics/profitability/per-project/{project_id}/lines`
- Por cliente: `GET /metrics/profitability/per-client/{client_id}/lines`

La respuesta clasifica cada línea en `incomes` o `expenses` según el signo de `amount`.

### Guardar snapshot

El botón **Guardar snapshot** guarda el estado actual de la vista de rentabilidad en la colección `chart_snapshots` de MongoDB (CU-17).

---

## Datos del resumen global

| Campo | Descripción |
|---|---|
| **income** | Suma total de ingresos del periodo (`amount >= 0`). |
| **expense** | Suma total de gastos (`amount < 0`). |
| **net** | `income + expense`. |
| **total_hours** | Suma de `unit_amount` (horas imputadas). |
| **profitability_pct** | `net / income × 100`. Si `income = 0` y `net < 0` → `-100 %`. |
| **status** | `ganancia` si `profitability_pct > PROFITABILITY_GAIN_THRESHOLD`, `neutro` o `perdida`. |
| **projects_summary** | Recuento `{ ganancia, neutro, perdida, total }`. |

---

## Restricciones de acceso

- **Solo Director.** El guard `require_director` rechaza con 403 cualquier token con rol distinto de `director`, antes de que la petición llegue al servicio.
- No se aplica filtrado por scope: la rentabilidad es un dato global de negocio que el Director consulta sin restricción organizativa.

### Capa de rutas — `routes/metrics.py`

Los tres endpoints de rentabilidad comparten el guard `require_director`. Los filtros `date_from`, `date_to`, `project_id` y `manager_id` son opcionales en todos ellos.

# Resumen global de ingresos / gastos / neto
@router.get("/profitability/summary")
def profitability_summary(

    date_to:   Optional[date] = Query(None),
    db: Session     = Depends(get_db),

### Capa de servicio — `services/metrics/rentability_service.py`

Los métodos auxiliares `_pct` y `_status` encapsulan la lógica de clasificación financiera y se reutilizan en las tres vistas. La agregación por cliente se realiza en memoria a partir del resultado por proyecto, sin una consulta SQL adicional.

```python
@staticmethod
def _pct(net, income):
    # Si no hay ingresos pero hay pérdida, devuelve -100 % por convención
    if income == 0:
        return -100.0 if net < 0 else 0.0
    return round((net / income) * 100, 2)

@staticmethod
def _status(pct, net):
    if net < 0:
        return "perdida"
    # PROFITABILITY_GAIN_THRESHOLD es una constante configurable (p.ej. 10 %)
    return "ganancia" if pct > PROFITABILITY_GAIN_THRESHOLD else "neutro"

def get_summary(self, date_from=None, date_to=None):
    row = get_global_totals(self.db, date_from, date_to)
    inc, exp, net, hrs = (float(getattr(row, a) or 0)
                          for a in ("income", "expense", "net", "total_hours")) if row else (0,0,0,0)
    pct = self._pct(net, inc)

    # Reutiliza la consulta por proyecto para el recuento de estados
    proj_rows = get_profitability_by_project(self.db, date_from=date_from, date_to=date_to)
    counts = {"ganancia": 0, "neutro": 0, "perdida": 0}
    for pr in proj_rows:
        counts[self._status(
            self._pct(float(pr.net or 0), float(pr.income or 0)),
            float(pr.net or 0)
        )] += 1

    return {"income": round(inc, 2), "expense": round(exp, 2), "net": round(net, 2),
            "total_hours": round(hrs, 2), "profitability_pct": pct,
            "status": self._status(pct, net),
            "projects_summary": {**counts, "total": sum(counts.values())}}

def get_per_client(self, date_from=None, date_to=None, project_id=None, manager_id=None):
    # Reutiliza get_per_project y agrega en memoria por (client_id, client_name)
    proj_rows = self.get_per_project(date_from, date_to, project_id, manager_id)
    clients = {}
    for p in proj_rows:
        key = (p["client_id"], p["client_name"] or "Sin cliente")
        if key not in clients:
            clients[key] = {"client_id": key[0], "client_name": key[1],
                            "project_count": 0, "income": 0, "expense": 0,
                            "net": 0, "total_hours": 0, "projects": []}
        c = clients[key]
        c["project_count"] += 1
        for field in ("income", "expense", "net", "total_hours"):
            c[field] += p[field]
        c["projects"].append({"project_id": p["project_id"], "project_name": p["project_name"],
                               "status": p["status"]})
    # Aplica _pct / _status sobre los totales agregados del cliente
    result = []
    for c in clients.values():
        pct = self._pct(c["net"], c["income"])
        result.append({**c, "profitability_pct": pct,
                        "status": self._status(pct, c["net"])})
    result.sort(key=lambda x: x["profitability_pct"])
    return result
```

### Capa de repositorio — `repositories/metrics/rentability.py`

El repositorio trabaja directamente sobre `account_analytic_line` (tabla de Odoo). Los ingresos y gastos se distinguen por el signo del campo `amount`. Tres expresiones SQL helper encapsulan los `CASE WHEN`:

```python
# repositories/metrics/rentability.py

def _income_expr():
    # Suma solo filas con amount > 0 (cobros facturados)
    return func.coalesce(func.sum(case((Timesheet.amount > 0, Timesheet.amount), else_=0)), 0)

def _expense_expr():
    # Suma solo filas con amount < 0 (costes de horas imputadas)
    return func.coalesce(func.sum(case((Timesheet.amount < 0, Timesheet.amount), else_=0)), 0)

def _net_expr():
    # Suma algebraica de todo: income + expense (expense ya es negativo)
    return func.coalesce(func.sum(Timesheet.amount), 0)

def _apply_filters(q, project_id, manager_id, date_from, date_to):
    # Excluye líneas sin proyecto o sin importe (presupuestos no facturados)
    q = q.filter(Timesheet.project_id.isnot(None), Timesheet.amount.isnot(None))
    if project_id:  q = q.filter(Timesheet.project_id == project_id)
    if manager_id:  q = q.filter(Timesheet.manager_id == manager_id)
    if date_from:   q = q.filter(Timesheet.date >= date_from)
    if date_to:     q = q.filter(Timesheet.date <= date_to)
    return q

def get_profitability_by_project(db, project_id=None, manager_id=None,
                                  date_from=None, date_to=None):
    # GROUP BY project_id → una fila por proyecto con ingresos / gastos / neto / horas
    q = db.query(
        Timesheet.project_id.label("project_id"),
        _income_expr().label("income"),
        _expense_expr().label("expense"),
        _net_expr().label("net"),
        func.coalesce(func.sum(Timesheet.unit_amount), 0).label("total_hours"),
    )
    q = _apply_filters(q, project_id, manager_id, date_from, date_to)
    return q.group_by(Timesheet.project_id).all()

def get_global_totals(db, date_from=None, date_to=None):
    # Sin GROUP BY → fila única con los totales globales del sistema
    q = db.query(
        _income_expr().label("income"),
        _expense_expr().label("expense"),
        _net_expr().label("net"),
        func.coalesce(func.sum(Timesheet.unit_amount), 0).label("total_hours"),
    )
    q = _apply_filters(q, None, None, date_from, date_to)
    return q.first()

def get_project_meta(db, project_ids):
    # JOIN con res_partner para obtener nombre de cliente por proyecto
    rows = (
        db.query(Project.id, Project.name, Project.partner_id,
                 Partner.name.label("partner_name"))
        .outerjoin(Partner, Project.partner_id == Partner.id)
        .filter(Project.id.in_(project_ids))
        .all()
    )
    return {
        r.id: {
            "name":         extract_translated_name(r.name) or f"Proyecto {r.id}",
            "partner_id":   r.partner_id,
            "partner_name": r.partner_name or "--",
        }
        for r in rows
    }
```
